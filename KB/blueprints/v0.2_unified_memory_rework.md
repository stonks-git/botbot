# Blueprint v0.2: Unified Memory — Discard L0/L1 Layers

## Decision

**D-005: Discard L0/L1 layered memory in favor of unified memory weights.**

The original intuitive-AI has two parallel identity systems:
1. L0/L1 JSON files (`layers.py`) — manually structured identity (name, persona, values, beliefs, boundaries, goals)
2. Unified memory table with Beta(alpha, beta) weights where high center = strong belief

In BotBot, identity IS the weights. The Beta distribution is the whole mechanism:
- Memory reinforced → high alpha → high center → surfaces more in stochastic sampling
- Memory contradicted → high beta → low center → surfaces less
- Identity emerges organically from what survives this weight evolution

The original's context_assembly Track 2 already pulls "top-20 identity memories by depth_weight center" from the DB — not from L0/L1 files. And Phase 5 consolidation's `_promote_patterns()` writes back to L0/L1 FROM the memory table. So L0/L1 is a derived cache, not a source of truth. Remove the cache; use the source directly.

**Rewrite — Gate wiring (lines ~317-323):**
```python
# WAS: LayerStore.get_all_layer_embeddings() from JSON files
# NOW: Top-N high-weight memory embeddings from DB

async def _get_identity_embeddings(agent_id: str, top_n: int = 20):
    """Top-N memories by weight center — these ARE the agent's identity signal for the gate."""
    store = _store()
    rows = await store.pool.fetch("""
        SELECT content, embedding,
               depth_weight_alpha / (depth_weight_alpha + depth_weight_beta) AS center
        FROM memories
        WHERE agent_id = $1 AND embedding IS NOT NULL
        ORDER BY depth_weight_alpha / (depth_weight_alpha + depth_weight_beta) DESC
        LIMIT $2
    """, agent_id, top_n)
    if not rows:
        return None
    result = []
    for r in rows:
        emb = np.frombuffer(r["embedding"], dtype=np.float32)  # halfvec → need to verify deserialization
        result.append((r["content"], r["center"], emb))
    return result or None
```

## Agent Core Metadata (name, persona, voice)

The one thing L0/L1 provided that unified memory doesn't is core metadata (name, persona, voice). These aren't memories that emerge from experience — they're configuration.

**Chosen approach: Immutable seed memories.**

At agent creation, seed 1-3 immutable memories:
- `memory_type="identity"`, `immutable=true`, high initial weight `Beta(8, 2)` (center ~0.8)
- Content: "My name is X", "My voice is Y", "My persona is Z"
- These always surface in identity render (immutable + high weight)
- They can never be forgotten (immutable flag) but CAN be further reinforced

This keeps everything in one table. No special-casing needed.
