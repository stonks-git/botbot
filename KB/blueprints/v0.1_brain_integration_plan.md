# Plan v2: Bolt intuitive-AI Cognitive Architecture onto OpenClaw

## Context

**Problem**: OpenClaw has a basic memory plugin (LanceDB vector store with simple save/recall). intuitive-AI has a full cognitive architecture вҖ” memory with evolving weights, gut feeling, consolidation (sleep), daydreaming (DMN), safety monitoring. The goal is to give OpenClaw a real mind by running the intuitive-AI brain as a Python sidecar service.

**Decisions made**:
- Architecture: Python sidecar (brain service) + thin TypeScript OpenClaw plugin
- Scope: Full brain (memory, gut, consolidation, DMN, safety, bootstrap)
- Database: PostgreSQL 17 + pgvector (replaces LanceDB)

## Glossary (plain English)

- **Brain service**: A Python program that runs alongside OpenClaw. It handles all the "thinking" вҖ” storing memories, feeling emotions, dreaming. OpenClaw talks to it over HTTP (like a web API on your local machine).
- **Consolidation**: NOT an agent/bot. It's a background process inside the brain service вҖ” like your brain processing the day while you sleep. Every few minutes it does light cleanup (fading unused memories, spotting contradictions). Every hour it does deeper work (finding patterns, generating insights, compressing old memories). It runs automatically inside the brain, not as a separate thing.
- **Multiple agents / agent_id**: OpenClaw can run different AI personalities вҖ” e.g., one bot for Telegram, another for Discord, another for work Slack. Each is an "agent." The brain adds an `agent_id` tag to every memory so each agent has its own separate brain. Your Telegram bot doesn't see what your Discord bot remembers (unless you want it to).
- **DMN (Default Mode Network)**: The "daydreaming" system. When an agent is idle (nobody talking to it), the brain randomly revisits old memories, makes creative connections, and can get stuck thinking about emotionally charged topics. Like shower thoughts.
- **Gut feeling**: A mathematical compass that compares "what I'm thinking about right now" to "everything I've ever experienced." Produces something like emotions вҖ” calm when things align, agitated when something unexpected shows up.
- **Entry/Exit gate**: A filter that decides whether incoming information is worth remembering. Not everything gets saved вҖ” boring/repetitive stuff gets dropped, novel/important stuff gets kept. Uses a psychology model called ACT-R.
- **Beta distributions**: Each memory's importance isn't a fixed number вҖ” it's a probability curve that captures both the strength AND the uncertainty. A new memory has wide uncertainty. A well-reinforced memory has tight certainty. This lets memories naturally "debate" each other.

**Plugin flow** (what happens when someone sends a message):
1. `message_received` hook вҶ’ brain updates attention centroid (tracks what agent is focused on)
2. `before_agent_start` hook вҶ’ brain assembles context (relevant memories + identity + gut state + any DMN shower thoughts) вҶ’ plugin injects into the LLM prompt
3. Agent runs with enriched context + cognitive tools (memory_store, memory_recall, gut_check, introspect)
4. `agent_end` hook вҶ’ plugin sends conversation to brain's entry gate (the filter that decides what's worth remembering)
5. Background: plugin polls brain for DMN thoughts every 30s, queues as low-priority self-prompts when agent is idle

## Key Design Decisions

1. **Embedding model**: Upgrade to `gemini-embedding-001` at **3072 dimensions** (max resolution). The original intuitive-AI used 768-dim which is the model's lowest Matryoshka truncation. Full 3072-dim gives 4x the resolution for more precise gut feeling deltas and finer-grained semantic search. The model supports Matryoshka truncation (768/1536/3072) so we can always reduce later if storage becomes a concern. DB schema uses `halfvec(3072)` for the embedding column.
2. **Agent namespacing**: Every DB table has `agent_id TEXT NOT NULL`. Resolved from OpenClaw's `ctx.agentId` in plugin hooks. Each agent gets its own separate memory, identity, and gut feeling.
3. **Layer state storage**: Filesystem at `brain-state/{agent_id}/` on the brain container's volume. Not exposed to OpenClaw.
4. **Graceful degradation**: If brain is unreachable, plugin skips context enrichment and logs a warning. OpenClaw keeps working without memory.
5. **Consolidation LLM**: Brain has its own LLM client for the background "sleep" processing (insight/narrative generation). Configurable provider. Main agent LLM calls go through OpenClaw вҖ” the brain never talks to users directly.
6. **Multi-agent consolidation**: Brain discovers active agents via `SELECT DISTINCT agent_id FROM memories`, runs background processing for each agent on independent schedules.
