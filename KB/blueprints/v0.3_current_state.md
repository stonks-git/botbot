## What BotBot Is

BotBot bolts the intuitive-AI cognitive architecture onto OpenClaw as a Python sidecar brain service, giving OpenClaw agents a real mind that develops identity from experience.

**Stack**: Python 3.12 (brain) + Node.js/TypeScript (OpenClaw) + PostgreSQL 17 + pgvector

## Core Design Principle: Identity IS the Weights (D-005)

There are no L0/L1 JSON files. No `layers.py`. Identity emerges from the unified `memories` table:

- Memory reinforced → high alpha → high weight center → surfaces more in stochastic sampling
- Memory contradicted → high beta → low weight center → surfaces less
- Identity = the top-N highest-weight memories. Values, beliefs, goals are just memories that survived reinforcement.
- Agent core metadata (name, persona) = immutable seed memories with `Beta(8,2)` and `immutable=true`

- `brain/src/stochastic.py` — StochasticWeight: Beta(alpha,beta) with observe/reinforce/contradict
- `brain/src/activation.py` — ACT-R: B+S+P+epsilon, cosine_similarity, spreading_activation
- `brain/src/relevance.py` — 5-component Dirichlet relevance, co-access (Hebbian), spread_activation
- `brain/src/memory.py` — MemoryStore: embed (Gemini), store, search_similar/hybrid/reranked, retrieval mutation, store_insight, why_do_i_believe

**API**: POST /memory/store, POST /memory/retrieve, GET /memory/{id}, DELETE /memory/{id}

**Plugin** (`openclaw/extensions/memory-brain/`):
- Tools: memory_recall, memory_store, memory_forget
- Hook `before_agent_start`: context assembly → inject via prependContext
- Hook `agent_end`: auto-capture user messages via gate (max 3/session)

### Phase 2: Entry/Exit Gate — DONE
- `brain/src/gate.py` — EntryGate (stochastic filter) + ExitGate (3x3 decision matrix)
- Entry: short (<10 chars) 95% skip, mechanical (prefixes) 90% skip, normal 99% buffer
- Exit: relevance (core/peripheral/irrelevant) × novelty (confirming/novel/contradicting) → persist/reinforce/buffer/drop
- Score modulation: `base_score * (0.5 + 0.5 * s_i) + emotional_charge_bonus`
- Noise floor: 2% chance DROP → BUFFER

**API**: POST /memory/gate

### Phase 3: Context Assembly + Unified Identity — DONE (D-005 rework)
- `brain/src/context_assembly.py` — 3-track context assembly:
  - Track 0: immutable safety memories (always present)
  - Identity hash: compact ~100-200 tokens from top-10 memories by weight center
  - Track 2: stochastic identity memories (Beta-sampled `observe() > 0.6` from top-20 by weight)
  - Track 1: situational memories (hybrid search within budget)
- `render_identity_hash(memory_store, agent_id)` — from DB, replaces LayerStore
- `render_identity_full(memory_store, agent_id)` — from DB, grouped by memory type
- `render_system_prompt()` → `[SAFETY BOUNDARIES]`, `[IDENTITY]`, `[IDENTITY -- active beliefs/values]`, `[RELEVANT MEMORIES]`
- Gate wiring: `_get_identity_embeddings()` in api.py queries top-20 memories by weight center, `embedding::float4[]` SQL cast for asyncpg deserialization → `np.array()`

**API**: POST /context/assemble, GET /identity/{agent_id}, GET /identity/{agent_id}/hash
**Plugin**: introspect tool (view identity, full or hash mode)

## Patterns Established

- **asyncpg halfvec deserialization**: Use `embedding::float4[]` SQL cast → returns Python list → `np.array(..., dtype=np.float32)`
- **Agent namespacing**: Every DB query includes `agent_id` filter. Every endpoint requires it.
- **Stochastic injection**: Use `StochasticWeight.observe() > threshold` for probabilistic context inclusion
- **Graceful degradation**: If brain unreachable, plugin logs warning and falls back (raw memory retrieval or no enrichment)
- **Weight center query**: `depth_weight_alpha / (depth_weight_alpha + depth_weight_beta)` — indexed in schema
